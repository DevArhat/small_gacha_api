# small_gacha_api

## 외환 유출 시뮬레이터
- - -
### 사용법

get /simulate/{game}/{target_rank}

post /simulate/

post를 하면 request body에다가 파라미터를 넣어서 요청하는거라던데

어케하는거지?

- - -
### {game} : str

/list 또는  
/list/aliases 에서 구현된 확률모델 목록 확인가능

list의 id로 지정된 항목 뿐 아니라
aliases에 들어있는 영문 모두 사용할 수 있음

맞는 항목이 없으면 기본값은 붕괴스타레일
- - -
### {target_rank} : int

돌파 기준으로 숫자 입력  
(이상한거 쓰면 에러뱉음, 음수는 0으로 맞추고 큰 수는 각 게임별 풀돌로 맞춤)

* 명함: 0
* 1돌: 1
* ...
- - -
### 응답 형태 예시

JSON 형식으로 반환

```json
      "[Pull 534] 5★ 픽업 획득 (4번 획득: 3돌)",
      "[Pull 612] 5★ 픽업 획득 (5번 획득: 4돌)",

```
raw -> pulls = 총 뽑기 횟수 (total_pulls랑 같은 값)

raw -> cost = 뽑기 비용 (뽑기 횟수 기준 추산)

after_exchange -> pulls = 총 뽑기 횟수에서 스타라이트, 잔류신호 등으로 교환 가능한 뽑기권 갯수를 뺀 값

after_exchange -> cost = 그 횟수를 기준으로 추산한 뽑기 비용

trucks -> raw, raw_cost = 총 뽑기 횟수 기준 필요 트럭 갯수, 트럭 기준 비용

trucks -> after_exchange, after_exchange_cost = 페이백 공제 기준 필요 트럭 갯수, 트럭 기준 비용

crumbs -> total = 페이백 뽑기권 교환재료 총 획득 갯수

crumbs -> tickets_changed = 그걸로 바꾼 뽑기권 수

crumbs -> remaining = 남은 페이백 뽑기권 교환재료 수

logs -> log = 5성 획득 로그

logs -> target = 픽업이 뜬 횟수 (원신에서 별빛포착으로 먹은 경우 숫자 앞에 R 붙음, 붕/젠은 그런건 딱히 없음)

### 기본값 관련 주저리...

원신이 기본값이었는데

별빛포착이 좀 복잡한데다 정확한 작동 방식이 알려지지 않았고, 지금 시뮬레이터 확률이 호요버스 공지보다 1%정도 높게 나와서 바꿈
(반천 픽업확률 약 56.2% 나옴, 호요버스 공지는 55%)

원신이랑 가장 확률설정이 비슷하면서 뽑기로직이 직관적이라서 붕스로 함
